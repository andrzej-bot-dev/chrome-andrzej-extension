Andrzej — Settings

Two modes: your own OpenClaw (e.g. on Raspberry Pi) or any LLM via API (like the Jan app — just need a key). Switch mode and model from the side panel.

LLM providers — add a key, pick a model

Enter a provider's API key — the address and format are pre-configured. Click "Fetch models" to pull the current list and select a model from the dropdown (no need to type names). Keys are stored locally in this browser.

Advanced: override API addresses

Leave empty to use defaults. Change only if a provider has changed their endpoint.

OpenClaw connection
E.g. ws://raspberrypi.local:18789 or ws://192.168.1.50:18789. The gateway must listen beyond loopback — see README.
Tried automatically when the local address doesn't respond. Publicly always wss:// (Cloudflare tunnel / reverse proxy / Tailscale) — instructions in README, "Remote access" section.
On the Raspberry Pi: openclaw config get gateway.auth.token (or the value from ~/.openclaw/openclaw.json).

One-time gateway configuration (Raspberry Pi)

The gateway checks the browser Origin header — you must add this extension's ID to the OpenClaw config once and approve the device. Details in README.
